# Приложение для прогноза погоды

## Задача
Необходимо создать веб-приложение с прогнозом погоды, в котором пользователь сможет:
* При первом открытии приложения, будет запрошен доступ к геопозиции пользователя, и будет выведена прогноз на сегодня, а так же минимум на 2 следующих дня (В интерфейсе не нужно указывать что это за локация, можно написать "Текущее местоположенее")
* При отклонении разрешения доступа к гео-данным, появится форма добавления города для получения прогноза погоды (с идентичными требованиями)
* У пользователя так же должна быть возможность добавить как минимум 2 других города для получения информации о погоде в них
* В интерфейсе должна быть предусмотрена кнопка "Обновить" по которой будет заново запрашиваться информация о погоде (искуственное обновление страницы не считается, нужно заново отправить запросы)
* При перезагрузке приложения, информация о текущем местоположении (или вручную введеном городе) должна остаться, как и дополнительные города которые ввел пользователь

## Технические требования
* Запрещается использовать CSS-фреймворки (Bootstrap, Foundation и т.д.) и JS-фреймворков (Angular, React, Vue и т.д.).
* Разрешается использование препроцессоров CSS (SCSS) по желанию

  * Ссылку на репозиторий

## Этапы разработки
### Макет и верстка
* Создать базовую HTML-страницу приложения, элементы необходимые для отображения информации о погоде (контейнеры, модальные окна и тд)
* Адативный дизайн и медиазапросы. Приложение должно корректно отображаться как на мобильных, так и на десктопных устройствах. не допускается наложение элементов интерфейса друг на друга

### Добавление логики
* Получение информации о прогнозе погоды, должно осуществляться при помощи http-запросов. Допускается использование любого инструмента для создания http-запросов (fetch, axios и тд)
* Приложение должно корректно отображать состояния загрузки, ошибки при осуществлении http-запросов.
* При добавлении города, у пользователя должен появляться выпадающий список при вводе текста (Этот список можно захардкодить, можно так же получать с какого-то бесплатного api). Пользователь может добавить только существующий город, при введении некорректного города, должна быть показана соответствующая ошибка (под полем ввода)
* Необходимо сохранять информацию, введеную пользователем, при обновлении страницы. (Запросы с информацией о погоде должны отправляться заново)

## Критерии оценивания

## Верстка и адаптивность - 3 балла
* 2 балл - Аккуратная базовая верстка, корректное отображение приложения, отсутствие наложения элементов интерфейса
* 1 балл - Наличие всех требуемых элементов интерфейса

## Геолокация и загрузка текущей погоды - 4 балла
* 2 балл - Корректное получение геолокации при первом открытии
* 1 балл - Корректная обработка отказа (форма ввода города)
* 1 балл - Пока прогноза погоды для городов (первой локации и последующих выбранных). Прогноз минимум на 3 дня (сегодня + 2)

## HTTP-запросы и обработка состояний - 4 балла
* 1 балл - Получение погоды через HTTP-запросы
* 2 балл - Реализованы состояния интерфейса: загрузка / ошибка / успех
* 1 балл - Кнопка «Обновить» выполняет новые запросы (не reload)

## Работа со списком городов — 3 балла
* 1 балл - Добавление минимум 2 дополнительных городов
* 2 балл - Выпадающий список, валидация города, отображение ошибок

## Сохранение состояния - 1 балл
* 1 балл - Сохранение данных через localStorage, корректное восстановление

## Бонус
Дизайн и расположение элементов полностью остаётся на усмотрение студента.
Дополнительно, три работы, которые преподаватели сочтут наиболее интересными, получат бонусные баллы.

## Штрафы
* Минимальная функциональность приложения - Запрос геолокации пользоваеля (при отклонении - ввод города вручную). Возможность добавления других городов и переключения между ними. Демонстрация прогноза погоды на сегодня + минимум на 2 дня вперед. В противном случае работа будет оценена в 0 баллов.
* Использование запрещённых фреймворков - работа оценивается в 0 баллов. Работа должна быть реализована с использованием только чистого HTML, CSS (препроцессоры ок) и JavaScript (допускается использование TypeScript).
